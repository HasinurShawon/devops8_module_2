Assignment on module 2
 Assignment
 Topic: Version Control using Git and GitHub – Repository Setup, Branching, and Merging

Steps:
1. Set Up a Local Git Repository

- Create a new folder for the assignment on your computer.

- Initialize a new Git repository in that folder.

- Create a file named README.md and write a short description of the project.

- Stage and commit the file with a clear commit message.


2. Create and Connect a GitHub Repository

- Create a new repository on GitHub with a suitable name.

- Connect your local repository to the GitHub repository using git remote add origin.

- Push your local main branch to GitHub.


3. Create and Work on a New Branch

- Create a new branch named feature-update.

- Switch to the feature-update branch.

- Edit README.md or create a new file (e.g., notes.txt) and add some additional content.

- Stage and commit your changes on the feature-update branch.


4. Push the Branch and Create a Pull Request

- Push the feature-update branch to GitHub.

- Open your repository on GitHub and create a Pull Request (PR) from feature-update into main.

- Write a brief description of the changes in the Pull Request.


5. Merge and Update Local Repository

- Merge the Pull Request into the main branch on GitHub.

- On your computer, switch back to the main branch.

- Pull the latest changes from GitHub to update your local repository.


6. Submission Instructions

- Make sure your GitHub repository is public.

- Submit the link to your public GitHub repository.

Deliverables
·  A public GitHub repository link that shows:

- The initial commit with README.md.

- A separate branch (feature-update) with additional changes.

- A Pull Request that has been merged into main.
